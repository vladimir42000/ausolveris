## Purpose

This document defines the governing principles of the project.

## Core rules

- The repository is the authoritative project memory.
- Major decisions must be written, not left in chat history.
- Architecturally significant changes require explicit review and usually an ADR.
- Stable documents must not be silently rewritten.
- The project prefers clear narrow scope over broad vague ambition.
- Validation and traceability are first-class concerns.
