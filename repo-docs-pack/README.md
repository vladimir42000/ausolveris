# Repository Documentation Pack

This pack contains the retained project documents reorganized into a shallow, numbered Markdown structure intended to become the first clean repository documentation tree.

## Folder map

- `docs/00-governance` — project rules, roles, planning and handover model
- `docs/10-discovery` — mission, feasibility, concept exploration, risks, scope
- `docs/20-architecture` — whole-tree architecture exploration and geometry concepts
- `docs/30-spec` — contracts, ownership, granularity and API-facing specs
- `docs/40-validation` — test strategy, benchmark and numerical conventions
- `docs/50-operations` — status, changelog and operational documents
- `docs/90-templates` — templates for repeatable project artifacts
- `docs/adr` — architecture decision records

## Notes

These documents are intentionally prepared as draft/proposed/accepted artifacts with visible ownership headers so they can be reviewed and then committed into a new git repository as the authoritative project memory.
