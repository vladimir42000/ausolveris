# V1 Scope

## Status

Draft for review. Not yet frozen.

## Scope philosophy

V1 must be narrower than the long-term ambition. It must establish a trustworthy backbone plus one useful modeling pathway.

## Provisional v1 objective

Deliver a first usable open-source backbone for YAML-defined loudspeaker studies together with one benchmark-backed early modeling pathway built on reusable domain and geometry abstractions.

## In-scope areas

- YAML project definition skeleton,
- project validation and normalization,
- core units and domain types,
- first reusable geometry and flare abstraction layer,
- one initial solver pathway,
- structured outputs and result metadata,
- benchmark-aware validation scaffolding.

## Likely out-of-scope areas

- broad all-purpose solver coverage,
- polished GUI,
- unrestricted hybrid workflows,
- multiple production-grade solver families in parallel,
- aggressive optimization before benchmarked need exists.

## Open scope decisions

- exact first problem family,
- exact first solver path,
- required v1 outputs,
- benchmark acceptance thresholds.

## Freeze condition

V1 is frozen only after owner review of feasibility, benchmark direction, and numerical conventions.
