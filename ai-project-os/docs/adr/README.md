# ADR Directory

This directory stores architecture decision records.

## Rules

- One decision per file.
- Use increasing identifiers such as `ADR-001`, `ADR-002`, and so on.
- Accepted ADRs are historical records and should not be rewritten silently.
- If a decision changes, create a new ADR and mark the older one as superseded.
- Link task packets and merge requests to relevant ADRs.

## Suggested starting ADRs

- ADR-001: Governance operating model.
- ADR-002: Python runtime and tooling.
- ADR-003: Initial solver scope.
- ADR-004: Documentation and append-only policy.
